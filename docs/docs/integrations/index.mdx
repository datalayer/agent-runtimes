# Integrations

Agent Runtimes provides a flexible agent architecture built on top of [Pydantic AI](https://ai.pydantic.dev).

For the full Agentspecs reference and platform documentation, see [datalayer.ai/docs](https://datalayer.ai/docs).

## Why Pydantic AI?

We chose Pydantic AI as our foundation because it provides:

- **Type Safety** — Full type checking with Pydantic models for inputs and outputs
- **Structured Outputs** — Reliable JSON responses from LLMs with validation
- **Tool Calling** — First-class support for function tools and MCP toolsets
- **Multi-Model Support** — Works with Anthropic, OpenAI, Google, Azure, and more
- **Production Ready** — Well-tested and actively maintained

## Creating an Agent

```python
from pydantic_ai import Agent
from agent_runtimes.mcp import get_mcp_toolsets

# Get pre-loaded MCP toolsets
mcp_toolsets = get_mcp_toolsets()

# Create an agent with MCP tools
agent = Agent(
    "anthropic:claude-sonnet-4-20250514",
    system_prompt="You are a helpful assistant with access to web search.",
    mcp_servers=mcp_toolsets,
)
```

### Custom Agent Deployment

```python
from agent_runtimes import AgentRuntimesApp
from pydantic_ai import Agent

agent = Agent(
        model='anthropic:claude-sonnet-4-5',
        system_prompt='You are a helpful assistant.',
)

app = AgentRuntimesApp()
app.add_agent(agent, name='my-agent', transport='ag-ui')
app.run(port=8000)
```

## Runtime Lifecycle Management

Use runtime hooks to launch or attach compute resources and auto-create agents:

```typescript
import { useAgentRuntime } from '@datalayer/agent-runtimes/lib/runtime';

const { isReady, endpoint, tools, launchRuntime } = useAgentRuntime({
    autoCreateAgent: true,
    agentConfig: {
        model: 'anthropic:claude-sonnet-4-5',
        systemPrompt: 'You are a helpful AI assistant.',
    },
});

await launchRuntime({
    environmentName: 'python-simple',
    creditsLimit: 100,
    type: 'notebook',
});
```

## Hooks Integration (Pydantic AI)

Agent Runtimes reuses Pydantic AI capability hook methods directly for tool approval and guardrail flows.

Common hook points:

- `before_tool_execute`
- `after_tool_execute`
- `on_tool_execute_error`
- `deferred_tool_calls`

### AgentSpec Hook Example

```yaml
tool_hooks:
    actor: "${USER}"
    audit_log_path: "/tmp/agent_runtimes_tool_approvals_audit.jsonl"
    before_tool_execute:
        - function: agent_runtimes.integrations.tool_policy:evaluate_tool_request
    after_tool_execute:
        - python: |
                print("success", payload.get("tool"), payload.get("status"))
    on_tool_execute_error:
        - python: |
                print("error", payload.get("tool"), payload.get("error_type"))
```

### Agent_Sudo Compatibility (Audit Logging)

`agent-runtimes` users can emit [Agent_Sudo](https://github.com/ram9199/agent_sudo)-compatible audit logs through existing tool hooks.

* **Ownership Boundaries**: `agent-runtimes` retains full ownership of the tool approval UX, hook lifecycle, and audit persistence. `Agent_Sudo` provides the interoperability specifications and verification tooling.
* **Verification**: User hooks (e.g. `after_tool_execute`) may emit records following the [Canonical Hash Chain Specification](https://github.com/ram9199/agent_sudo/blob/main/docs/design/CANONICAL_HASH_CHAIN.md) and [Policy/Audit Schema](https://github.com/ram9199/agent_sudo/blob/main/docs/design/POLICY_AUDIT_SCHEMA.md) (standardized in [v0.4.0-rc13](https://github.com/ram9199/agent_sudo/releases/tag/v0.4.0-rc13)).
* **Verifier Command**: Compatible logs can be verified at any time using the `agent-sudo` CLI:
  ```bash
  agent-sudo verify-audit path/to/mcp-audit.jsonl
  ```

For full specifications, refer to the [Interoperability Checklist](https://github.com/ram9199/agent_sudo/blob/main/docs/architecture/INTEROPERABILITY_CHECKLIST.md).


## Model Providers

Agents can use models from multiple providers:

| Provider | Model Format |
|----------|-------------|
| Anthropic | `anthropic:claude-sonnet-4-20250514` |
| OpenAI | `openai:gpt-4o` |
| Azure OpenAI | `azure:gpt-4o` |
| AWS Bedrock | `bedrock:us.anthropic.claude-3-sonnet` |
| Google | `google:gemini-1.5-pro` |

## Per-Request Model Selection

Switch models dynamically without restarting:

```python
# Use Claude for complex reasoning
result = await agent.run("Analyze this data...", model="anthropic:claude-sonnet-4-20250514")

# Use GPT-4 for creative tasks
result = await agent.run("Write a poem...", model="openai:gpt-4o")
```
